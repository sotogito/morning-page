# MVP

일기글 저장은 깃허브로 레퍼지토리에 저장한다.
통꼐에 필요한 데이터는 supagase에 저장한다.

### 0. 로그인
- 사용자는 아래와 같은 정보를 입력한다.
    - 레퍼지토리 링크
    - Access 토큰


### #1. 글작성 에디터
1. 기본 에디터
    - 사용자는 글을 작성할 수 있다.
    - 글 입력창은 단일 페이지 에디터로 구성된다.

2. 파일 읽기 (Markdown)
    - 시스템은 .md 파일만을 불러와 내용을 표시할 수 있다.
    - 파일 내용은 Markdown 포맷으로 파싱된다.

3. 제목 자동 생성 규칙
    - 새 글 작성 시 기본 제목은 오늘 날짜 YYYY-MM-DD    [제목] 형식이다.
    - 사용자가 제목을 직접 입력할 수 있다.
    - 제목이 입력되지 않으면 기본 날짜로 저장한다.
    - 날짜 생성 시, 날짜 뒤에 공백(space + 1)이 추가된다.
    - 저장 시에는 공백을 자동으로 trim 처리한다.
    - 사용자게에는 md 파일 확장자를 보이지 않는다.

4. 작성 시간 제한
    - 글 작성은 시작 시점부터 30분 동안만 가능하다.
    - 30분이 경과하면 저장되고 입력이 비활성화된다.

5. 백스페이스 제한
    - 작성 중에는 백스페이스 키 입력이 차단된다.
    - 사용자는 글자를 삭제할 수 없다.

6. 저장 조건
    - 글자 수가 1,000자 이상일 때만 저장이 가능하다.
    - 1,000자 미만이면 “작성량 부족” 경고 메시지를 표시한다.

7. 수정 제한 (자정 기준)
    - 한번 저장된 글은 수정할 수 없다.

### #2. 깃허브 연동
- 파일 읽어오기
- 파일 계층 띄우기 (옵시디언 처럼)
- 잔디심기 - commit api의 commit.author.date
- 잔디 누르면 해당 글로 이동
