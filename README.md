## 프롤로그

20살 때 처음 타투이스트로 일을 시작했습니다. 1인 1도안이라는 암묵적인 규칙 속에서 늘 창조성에 대한 갈망이 있었습니다. 22살, 줄리아 카메론의 『아티스트 웨이』를 처음 마주했고, 공책에서 노션으로, 노션에서 옵시디언으로 옮겨가며 이어온 저의 기록 여정을 함께 나누고자 합니다.

## **줄리아 카메론의 『아티스트 웨이』 모닝페이지**

모닝페이지는 줄리아 카메론의 『아티스트 웨이』에서 소개된 창조성 회복 훈련입니다. 저자는 10년이 넘는 기간 동안 꾸준히 실천해왔으며, 전 세계 수많은 아티스트들도 이 방법을 자신의 작업 과정에 적용하고 있습니다.

- 잘못 쓴 모닝 페이지란 없다.
- 모닝페이지는 창조성 회복의 실마리가 되는 도구이다.
- 모닝페이지에 다른 선택의 여지는 없다. 모닝페이지를 거르거나 줄이면 안 된다.
- 세 쪽을 가득 채울 때까지 무슨 말이든 쓰는 것이다.

## 모닝페이지 약속

1. 아침에 일어나 바로 작성합니다.
2. 작성 중인 글은 삭제할 수 없습니다. 오타도 괜찮습니다.
3. 저장된 파일은 수정, 삭제가 불가합니다. 생각을 수정할 필요 없습니다.
4. 1,000자 이상 작성해야 저장할 수 있습니다. 너무 깊은 생각을 할 필요 없습니다. 아무 말이나 작성해도 좋습니다.
5. 하루에 한 번만 작성할 수 있습니다. 날짜는 자동으로 구분됩니다.

# 시작하기

### 저장소 생성

1. 모닝페이지를 기록할 전용 저장소를 **private**으로 생성합니다.
2. 다음 템플릿을 다운로드하고, 저장소에 `.morningpage` 폴더를 추가합니다.
    
    `템플릿 : https://github.com/sotogito/morningpage-template.git`
    

### 로그인

1. 생성한 저장소의 URL과 GitHub Personal Access Token(PAT)을 사용해 로그인합니다.

# 제공 기능

### 1) 에디터

- 저장소 내 `.md` 파일을 읽어오고, 모닝페이지 양식에 맞는 파일만 표시합니다.
- 기본 폴더 구조는 다음과 같습니다.
    
    ```markdown
    1월
     └── 1째주
           ├── 2025-01-01 모닝페이지 첫 시작.md
           └── 2025-01-02.md
    ```
    
- 기본 제목은 오늘 날짜로 자동 생성되며, 다음 형태를 지원합니다.
    - `YYYY-MM-DD`
    - `YYYY-MM-DD 제목`
- 마크다운을 적용한 미리보기 기능을 제공합니다.

### 2) 즐겨찾기

- `.morningpage/favorites.json` 파일에 저장됩니다.
- 예: `1월/1째주/YYYY-MM-DD` 형태로 등록합니다.

### 3) 통계

- 기여도(히트맵)
    - 날짜별 작성 여부를 시각적으로 확인할 수 있습니다.
    - 자신의 모닝 타임을 설정할 수 있습니다.
        - `.morningpage/morning-time.json` 파일로 관리됩니다.
    - 설정한 모닝 타임에 따른 색상이 보여집니다.
        - 초록, 주황, 빨강
    - 특정 칸을 클릭하면 해당 파일로 이동합니다.
- 작성 통계
    - `.morningpage/status.json` 파일로 관리됩니다.
    - 총 작성일, 연속 작성일, 마지막 작성일을 제공합니다.

### 4) AI를 활용한 나의 창조성과 대화하기

모닝페이지는 개인의 창조성을 확장하기 위해 AI와 연결할 수 있는 파일 구조를 제공합니다.

- `.morningpage/ai/system.txt`
    
    기본 프롬프트이며, 수정은 권장하지 않습니다.
    
- `.morningpage/ai/instructions.md`
    
    AI에게 전달되는 지침서입니다.
    
- `.morningpage/user_prompt.md`
    
    사용자가 직접 편집해 추가 프롬프트를 설정할 수 있는 파일입니다.
    

### 사용 방법

1. 모닝페이지 파일을 로컬에 저장합니다.
2. AI 툴에서 파일을 불러옵니다.
3. AI와 대화를 시작합니다.

> 전체 파일 구조를 기반으로 작동하는 AI가 필요합니다.
Cursor, Claude Code 등을 권장합니다.

# 기타 안내

- 모닝페이지는 서버리스 구조로 동작합니다.
- 보안을 위해 로그인 정보를 브라우저에 저장하지 않습니다. 새로고침 시 다시 로그인이 필요합니다.
- GitHub PAT 기반으로 로그인하며, 모든 기록은 사용자의 개인 GitHub 저장소에 저장됩니다.
- 템플릿(.morningpage) 없이도 사용 가능하지만, 즐겨찾기 및 일부 통계 기능이 제한됩니다.
- 연도별로 저장소를 따로 관리하는 방식을 권장합니다.
